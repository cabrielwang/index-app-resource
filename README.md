# index-app-resource

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态资源的集合与分发点。

## 项目简介

本项目旨在提供一个集中存放 HTML 页面及相关资源的空间。每个页面均为独立、自包含的静态文件，适合快速部署、测试或分发。仓库结构清晰，便于索引与维护。

## 目录说明

```
index-app-resource/
├── pages/           # 存放各个独立 HTML 页面文件
├── assets/          # 页面引用的公共静态资源（如 CSS、JS、图片等）
├── docs/            # 文档与使用说明（可选）
└── README.md        # 本文件
```

- `pages/`：每个子目录或文件对应一个独立的 HTML 页面，命名建议反映页面功能或用途。
- `assets/`：存放页面共享的资源文件，避免重复引用。
- `docs/`：可放置额外的说明文档、变更日志等。

## 页面归档说明

- 所有 HTML 页面均为独立、自包含的静态文件，可直接在浏览器中打开。
- 页面之间无依赖关系，可单独使用或组合部署。
- 归档遵循“一个页面一个文件”原则，便于索引与版本管理。
- 资源引用路径建议使用相对路径，以确保在不同部署环境下正常工作。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 HTML 文件或目录，并更新本 README 中的页面列表（可选）。
- 更新页面：直接修改对应文件，建议保留变更记录。
- 删除页面：移除对应文件或目录，并清理相关引用。
- 资源管理：公共资源请放入 `assets/`，避免在页面内重复内联。
